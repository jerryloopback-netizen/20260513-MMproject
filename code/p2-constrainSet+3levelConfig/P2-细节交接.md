# P2 细节交接

## 目标

你负责项目入口的两步：

1. 硬约束过滤；
2. 录取概率估计与冲、稳、保、垫分档。

你输出的是“合法且已分档”的候选志愿集。P3 后续的 Pareto、语义抽取、多样性选择和整表生成，都依赖你的结果。

## 你在整体流程中的位置

整体流程：

1. 你先过滤硬约束；
2. 你再给候选志愿分档；
3. P3 在你的分档结果上做档内 Pareto 和语义抽取；
4. P1 提供权重，供 P3 在档内评分时使用。

## 硬约束原则

硬约束是“一票否决”，不能进入后续优化。

建议硬约束类别：

- 选科要求；
- 用户明确排斥专业；
- 地域限制；
- 学费或预算限制；
- 院校类型限制；
- 批次限制；
- 体检或单科限制；
- 招生计划面向省份限制；
- 数据严重缺失；
- 政策性限制。

## 规则库建议

### 政策规则

- 选科是否满足；
- 是否符合批次；
- 是否受体检限制；
- 是否存在单科最低要求。

### 用户规则

- 排斥专业；
- 排斥城市或省份；
- 学费上限；
- 不接受中外合作；
- 不接受民办；
- 不接受明显高风险调剂。

### 数据质量规则

- 录取概率缺失；
- 历史录取线缺失严重；
- 招生计划波动无法估计；
- 专业代码无法匹配；
- 数据口径不一致。

对数据质量问题，不一定全删，可标记 `review_required=true`。

## 分档思路

建议同时考虑点估计和下界：

\[
p_i^- = p_i - k\sigma_i
\]

其中：

- `p_i` 是录取概率点估计；
- `sigma_i` 是不确定性；
- `k` 是保守系数。

如果没有显式方差，可以用历史录取位次波动、招生计划变化、专业热度波动构造近似不确定性。

## 四档语义建议

| 档位 | 含义 | 处理方式 |
|---|---|---|
| 冲 | 有机会但不稳定 | 点估计可以低一点，但匹配和价值要高 |
| 稳 | 机会较大 | 平衡概率与价值 |
| 保 | 风险较低 | 更看重下界和稳定性 |
| 垫 | 极强安全性 | 主要追求兜底能力，但不能违背用户硬偏好 |

建议不要只凭单点概率机械切档，而是综合：

- `admit_probability`
- `probability_lower`
- `volatility_score`
- `adjustment_risk`
- `user_acceptance`

## 你要交付的内容

建议输出：

- 硬约束规则库；
- 过滤流程；
- 分档阈值；
- 概率下界处理；
- 异常和缺失数据处理；
- 输出字段说明；
- 至少 10 个测试样例。

测试样例建议覆盖：

- 正常通过；
- 选科不满足；
- 用户排斥专业；
- 概率高但调剂风险高；
- 点估计高但下界低；
- 数据缺失需人工复核；
- 保底不足；
- 极端安全垫底；
- 跨档边界；
- 多个硬约束同时失败。

## 和 P3 的衔接

P3 会基于你的输出做档内筛选与组合，所以你要保证：

- 档位标签稳定；
- 概率字段可复用；
- 失败原因可解释；
- 不要把不合法志愿混入候选集；
- 不要把硬约束和软偏好混为一谈。
