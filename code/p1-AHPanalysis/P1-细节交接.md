# P1 细节交接

## 目标

你负责项目里的 AHP 个性化权重模块。这个模块不是最终排序器，而是给后续“档位语义候选抽取”提供权重参数。

换句话说：

- 你要回答“用户更看重什么”；
- 不是直接回答“最终该报哪几个”。

## 你在整体流程中的位置

整体流程：

1. P2 做硬约束过滤和分档；
2. P1 输出用户个性化权重和四档修正权重；
3. P3 用你的权重在每一档内部做语义评分、候选抽取和多样性选择；
4. P3 最后做整表组合和蒙特卡洛评估。

## 推荐指标体系

建议一级指标至少包含：

- 专业匹配度 `M_fit`
- 就业发展价值 `E_career`
- 城市发展价值 `C_city`
- 用户软偏好 `U_pref`
- 录取安全性 `P_safe`
- 综合风险控制 `R_risk`

可选二级拆分：

- `M_fit`：兴趣匹配、学科能力匹配、排斥程度反向指标；
- `E_career`：薪资、岗位数量、稳定性、深造、行业成长性；
- `C_city`：产业机会、生活成本、离家距离、发展空间；
- `U_pref`：院校层次、地域偏好、专业偏好、家庭资源偏好；
- `P_safe`：点估计概率、保守概率下界；
- `R_risk`：调剂风险、冷热波动、计划波动、数据不确定性。

## AHP 的正确用法

AHP 只在“指标重要性”上发挥作用，不直接代替排序。

判断矩阵记为：

\[
A=(a_{ij})
\]

满足：

\[
a_{ij}=1/a_{ji}, \quad a_{ii}=1
\]

权重向量记为：

\[
w=(w_1,\dots,w_n)
\]

可以用特征向量法，也可以用几何平均法。

一致性检验必须保留：

\[
CI=\frac{\lambda_{\max}-n}{n-1},\quad CR=\frac{CI}{RI}
\]

通常要求：

\[
CR<0.1
\]

## 四档修正权重

用户基础权重记作 `w_user`，档位修正系数记作 `rho_tier`，则：

\[
w_j^{tier}=\frac{w_j^{user}\rho_j^{tier}}{\sum_k w_k^{user}\rho_k^{tier}}
\]

建议档位倾向如下：

- 冲：降低安全性权重，提高匹配、就业、城市和偏好；
- 稳：各项较均衡；
- 保：提高安全性和风险控制；
- 垫：安全性和风险控制优先，但不能违背硬偏好。

## 和 P3 的衔接

P3 会用你的权重做档内语义评分：

\[
S_i^{tier}=\sum_j w_j^{tier} z_{ij}
\]

其中 `z_ij` 是志愿 `i` 在指标 `j` 上的标准化值。

注意：

- 这个分数只在同一档内用；
- 不要跨档直接比较；
- 不要把它写成最终整表最优目标。

## 风险提醒

- 不要把 AHP 写成“最后自动选志愿”；
- 不要让判断矩阵过于复杂；
- 不要忽略一致性检验；
- 不要把档位修正系数写成硬编码真理，最好说明是语义参数。
